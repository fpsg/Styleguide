Buttons

Buttons are one of the most important elements on a website. If implemented correctly, buttons help us reach our business goals and the user reach their goals.

Primary Buttons

Primary buttons are reserved for the main interaction for a page. There can’t be more than one primary button per page, because there are not ever more than one primary action per page. In the checkout flow, proceeding to the next page is the primary (desired) action, therefore that button is the primary button. In the account section, the main action is for the user to save his/her changes, therefore the Save button becomes the primary button. Actions such as “next”, “continue”, “save” and “confirm” are essentially always primary actions, and should be primary buttons.

Secondary Buttons

A secondary button is used together with a primary one. The reason to why it’s a secondary button and not a second or third primary one, is because it’s less important than the primary button, but it still relates to the hierarchy somehow. If there’s a primary button in the layout, all other buttons become secondary (or tertiary). No matter if they are right next to the primary button on the page or far away from it.

Tertiary Buttons (Name change will come)

Tertiary buttons are for actions which don’t fit into a priority order. While sending an email is the primary action for an email editor, changing font color and size doesn’t have any priority but are still offered to the user. Liking and commenting on facebook is also a good example of actions where one is not prioritized against the other.

Microcopy/button text

In order to make the buttons as effective as possible, please write with a clear language and intent. Short phrases or single words, especially with verbs, help make the result of the action clear. “Delete Item” and “Sign Up” are two good examples, while “Confirm Action” and “Get it Now” are examples of bad ones.

Delete Buttons

Delete buttons, called so because they’re used to “delete” things, are essentially the primary/secondary/tertiary buttons with red instead of green. The actions performed with delete buttons are always destructive and maybe most importantly, not undoable. Even though you may display a warning/confirmation, for example after pressing the delete button there might be a popup asking the user to confirm the action, the action that you initiate with the delete button is always something destructive and non-undoable.

Combining Buttons

Only use one primary button per page, if you need to use it multiple times then the hierarchy isn’t clear enough on the page and the page probably has other problems that are not directly relatable to the button hierarchy. If you have buttons that are standing together with the primary button, they immediately become secondary buttons. It’s fine to mix two different hierarchies with each other, such as primariy with secondary, secondary with tertiary, and primary with tertiary, but not to use all three levels at once.
